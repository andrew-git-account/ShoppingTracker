# Data Schema — Shopping Tracker

Defines the data contracts for storage and LLM communication.

---

## Receipt JSON (stored in `data/receipts.json`)

The file contains a JSON array. Each element is one receipt.

```json
[
  {
    "id": "uuid-string",
    "upload_date": "2026-05-07T15:42:00",
    "store_name": "Walmart",
    "purchase_date": "2026-05-07",
    "currency": "USD",
    "items": [
      {
        "name": "Milk",
        "price": 3.99,
        "quantity": 1,
        "category": "Food & Groceries"
      }
    ],
    "subtotal": 3.99,
    "tax_amount": 0.32,
    "discount_amount": 0.00,
    "total_amount": 4.31,
    "is_deleted": false
  }
]
```

### Field reference

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | string (UUID) | Yes | Generated by the system on save |
| `upload_date` | string (ISO 8601 datetime) | Yes | When the receipt was processed |
| `store_name` | string | Yes | Extracted by LLM; empty string if unknown |
| `purchase_date` | string (YYYY-MM-DD) | Yes | Extracted by LLM; empty string if unknown |
| `currency` | string (ISO 4217) | Yes | Defaults to `"USD"` if LLM cannot determine |
| `items` | array | Yes | At least one item required |
| `items[].name` | string | Yes | Product name |
| `items[].price` | number | Yes | Price per unit |
| `items[].quantity` | integer | Yes | Minimum 1; weight-based items round up to 1 |
| `items[].category` | string | Yes | Must be from the predefined category list; defaults to `"Other"` |
| `subtotal` | number | Yes | Sum of items before tax/discount |
| `tax_amount` | number | Yes | Tax applied; 0.0 if none |
| `discount_amount` | number | Yes | Discount applied; 0.0 if none |
| `total_amount` | number | Yes | Final amount paid |
| `is_deleted` | boolean | Yes | `false` by default; set to `true` on soft delete |

## Categories (stored in `data/categories.json`)

```json
[
  { "id": 1, "name": "Food & Groceries" },
  { "id": 2, "name": "Household & Cleaning" },
  { "id": 3, "name": "Personal Care & Health" },
  { "id": 4, "name": "Electronics & Tech" },
  { "id": 5, "name": "Clothing & Apparel" },
  { "id": 6, "name": "Dining & Takeout" },
  { "id": 7, "name": "Other" }
]
```

This file is seeded automatically on first run if it does not exist. It is never overwritten if it already exists.

## LLM Response Contract

The LLM extraction prompt expects a JSON response in this shape:

```json
{
  "store_name": "Walmart",
  "purchase_date": "2026-05-07",
  "currency": "USD",
  "items": [
    {
      "name": "Milk",
      "price": 3.99,
      "quantity": 1,
      "category": "Food & Groceries"
    }
  ],
  "tax_amount": 0.32,
  "discount_amount": 0.00,
  "total_amount": 4.31
}
```

### Validation rules applied to the LLM response

| Field | Rule |
|---|---|
| `currency` | Must be uppercase ISO 4217; defaults to `"USD"` if missing or unrecognized |
| `items[].quantity` | Parsed as `max(1, int(float(value)))`; fractional quantities (weight-based items) round up to 1 |
| `items[].category` | Must match a name in the predefined category list; defaults to `"Other"` if missing or unrecognized |
| All numeric fields | Treated as 0.0 if missing |
| `store_name`, `purchase_date` | Stored as empty string if missing |
